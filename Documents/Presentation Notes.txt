Aviel: What is Taskademia:
- We sought to solve the problem of how to organize groups to work together. 
- Groups could organize goals
- Assign or pick up tasks
- See who's done what.

Aviel: Core Features:
- Projects
  - Public or private, join settings and recruiting status
- Kanban Task Board
- Team Management
  - Self-joining or approve new members
  - Member permissions (default to create and edit tasks),
  - Other permissions: manage other members or project details
- Notifications
  - tasks or projects change status 
  - when someone joins, leaves, or requests membership
- User Profile
  - profiles offer a way to tell others about yourself and your strengths

Aviel: Authentication System:
- Split the JWT into 2 components
  - Access Token you get on login and is verified on protected API routes
  - Refresh Token makes it easier to come back to the site

Manny: Tech Stack (going to sound like a dense list, try to make combine or skip some?)
- Front end:
  - Vite: Front end build tool. Helped development with quick start up, UI changes showed up instantly
  - Tailwind CSS: Handled most of our styling. Added responsiveness to the UI
  - TanStack: Helped with routing and organizing the site structure
  - Shadcn/ui: library of well-designed components
- Back end:
  - Node.js + Express: Handles routing, authentication, and communication with front end
  - JWT: manage secure login sessions and protect the routes and user data
  - AWS SES: Amazon Web Services, Simple Email Service - used for account verification, password reset. Highly scalable if we wanted to use email notifications
- Database: 
  - MongoDB: gave us a lot of flexibility, changed schemas as the project grew
- DevOps:
  - GitHub Actions: automated linting and having server pull new commits
  - PM2: rebooted backend after new commits

Luciano: How Things Went
- Went Well:
  - Figma prototype gave us something to work towards

  - By the end of spring break: we had most of backend done, core pages in front end working
  - AWS Email application went smoothly and got to production status in 24 hours.

- Did not go well:

  - Front end required a lot more work than we thought. Project details page and home page kept needing to be revisited to show everything a team would need (like the charts and invites)
  - As we expanded and updated the features the backend had to change and we needed to update the API documentation a lot more

Tim: Entity Diagram & Activity Diagram

Aviel: Class Diagram and Unit Tests
- Class Diagram:
  - This is a trimmed down class diagram. It shows how the mobile app is organized into three main parts: screens, services, and domain models. The screens handle the user interface. The service layer manages the API calls and token storage. And the domain models represent the app's main data like projects, tasks, and goals.
- Testing:
  - Our unit tests do not cover everything. We focused on the main backend controllers for authentication, projects, and tasks, and in the front end unit tests check utilities like JWT parsing, token refresh, and state management. The integration tests cover front and back user flows like registration, logging in and getting to the home page, a user getting and accepting an invitation, and viewing a project to submitting a join request.
